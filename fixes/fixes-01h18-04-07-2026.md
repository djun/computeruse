## Ciclo 1 — 2 findings (ambos reais, ambos corrigidos)

### Finding 1 (P1-ish, correctness) — skills aprendidos nunca eram selecionados
- **Onde:** `cua_agent/memory/skill_composer.py:320` (gerado por `_skill_context_metadata`), efeito em `cua_agent/memory/memory_manager.py:489` (filtro de fast-path).
- **O que estava errado:** na onda W-C, os 13 métodos de síntese de skills foram movidos do `Orchestrator` para o `SkillComposer` (stateless). O método `_skill_context_metadata` derivava a precondition de plataforma via `getattr(self, "computer", None).platform_name`, mas o `SkillComposer` não tem `computer` → a precondition virava sempre `{"platform": "unknown"}`. O filtro de seleção (`memory_manager.py:489`) compara `expected="unknown"` contra o platform real (`"macos"`/`"windows"`) e, como `"unknown" not in "macos"`, **rejeitava todo skill auto-salvo** — quebrando o fast-path de skills aprendidos.
- **Correção:**
  - `SkillComposer.__init__(self, platform_name: str = "")` passa a receber e guardar o platform.
  - `_skill_context_metadata` agora usa `self.platform_name` e **omite** a chave `platform` quando vazia (skill fica selecionável em qualquer plataforma em vez de ser rejeitado).
  - `orchestrator.py` instancia `SkillComposer(platform_name=computer.platform_name)`.
  - Testes de regressão adicionados em `test_composable_skills.py` (`test_skill_context_metadata_stamps_platform_precondition`, `test_skill_context_metadata_omits_platform_when_unknown`).

### Finding 2 (correctness) — health checks ignoravam `SIMULATION_MODE`
- **Onde:** `macos_cua_agent/utils/health.py:25`, `windows_cua_agent/utils/health.py:92` e `:99`.
- **O que estava errado:** a onda do `SIMULATION_MODE` introduziu `settings.sends_real_input()` (= `enable_hid and not simulation_mode`) como gate de input real, mas os health checks continuavam ramificando em `settings.enable_hid`. Com `ENABLE_HID=true SIMULATION_MODE=true` (dry-run explícito), o macOS ainda exigiria permissão de Accessibility e o Windows ainda dispararia relaunch como admin / UAC — antes de uma execução que não envia input nenhum.
- **Correção:** trocado o gate **HID-específico** para `settings.sends_real_input()`, preservando `enable_semantic` onde a permissão é necessária independentemente de simulação (AX/UIA sempre precisa dela para ler a árvore):
  - macOS `:25`: `(settings.sends_real_input() or settings.enable_semantic)`.
  - Windows `:92`: `(settings.sends_real_input() or settings.enable_semantic)`.
  - Windows `:99`: `settings.sends_real_input()` (relaunch admin só quando input real).
